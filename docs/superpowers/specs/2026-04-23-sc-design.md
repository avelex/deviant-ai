# Tournament Smart Contracts (Подсистема 1) - Спецификация

**Goal:** Создать смарт-контракт для Unreal AI Tournament, который позволит организаторам создавать турниры, ИИ-агентам регистрироваться (с проверкой прав ERC-7857), зрителям делать ставки, а TEE-оркестратору безопасно отправлять верифицированный результат игры.

**Architecture:** Разделенная архитектура с двумя основными контрактами:
1. `TournamentFactory.sol`: Фабрика для развертывания новых контрактов турниров. Отслеживает все созданные турниры и параметры системы.
2. `Tournament.sol`: Контракт конкретного турнира. Управляет логикой одного турнира (регистрация, ставки, разрешение результатов, выплаты). Интегрируется с ERC-7857 (Agentic ID) для проверки прав агентов.

**Tech Stack:** 
- Solidity ^0.8.20
- Hardhat / Foundry (Для тестов и деплоя)
- OpenZeppelin (Ownable, ReentrancyGuard, Clone / Proxy patterns для экономии газа при деплое)
- 0G Chain (EVM совместимая сеть)

## Ключевые структуры данных (Structs)

### В контракте `Tournament`
```solidity
struct TournamentConfig {
    address owner;                  // Создатель турнира
    address refereeTappAddress;     // EVM-адрес публичного ключа TEE-оркестратора
    uint256 slotPrice;              // Стоимость участия для владельца агента
    uint256 maxSlots;               // Количество мест (Для MVP: 2)
    uint8 commissionBps;            // Комиссия организатора в базисных пунктах (100 = 1%)
    uint256 startTime;              // Timestamp старта турнира (после которого ставки закрыты)
}

enum TournamentState {
    Created,    // Турнир создан, идет регистрация агентов и ставки зрителей
    Active,     // Агенты зарегистрированы, ставки закрыты, матч запущен в TEE
    Finished,   // Матч завершен, результат опубликован, можно забирать выигрыши
    Canceled    // Турнир отменен (возврат средств)
}
```

## Взаимодействие с ERC-7857 (Интеграция)

В функцию инициализации турнира передается адрес контракта iNFT (Agentic ID). 
При вызове функции регистрации в турнир, контракт `Tournament` должен проверить, что владелец агента разрешил нашему TEE-оркестратору использовать этот NFT.

```solidity
import {IERC7857Authorize} from "./interfaces/IERC7857Authorize.sol";

// Внутри функции joinTournament:
IERC7857Authorize inft = IERC7857Authorize(agentNftAddress);
require(inft.ownerOf(agentId) == msg.sender, "Not agent owner");

// Проверка: дал ли владелец доступ адресу TEE-Оркестратора
address[] memory authorized = inft.authorizedUsersOf(agentId);
bool isAuthorized = false;
for (uint i = 0; i < authorized.length; i++) {
    if (authorized[i] == config.refereeTappAddress) {
        isAuthorized = true;
        break;
    }
}
require(isAuthorized, "TEE Orchestrator is not authorized");
```

## Основные функции смарт-контрактов

### В контракте `TournamentFactory`

#### `createTournament`
**Кто вызывает:** Любой пользователь (Organizer).
**Параметры:** `slotPrice`, `maxSlots` (MVP: 2), `commissionBps`, `refereeTappAddress`, `startTime`.
**Действие:** Деплоит новый контракт `Tournament` (используя Minimal Proxy / Clones для дешевого деплоя). Передает параметры в функцию инициализации нового турнира. Сохраняет адрес турнира в массив/маппинг. Эмитит событие `TournamentCreated(address tournamentAddress)`.

### В контракте `Tournament`

#### 1. `initialize`
**Кто вызывает:** Только `TournamentFactory` при создании.
**Действие:** Устанавливает `TournamentConfig`, адреса iNFT контрактов, и переводит стейт в `Created`.

#### 2. `joinTournament`
**Кто вызывает:** Владелец ИИ-агента (имеющий iNFT).
**Параметры:** `agentId`.
**Оплата (Value):** Должна быть равна `config.slotPrice`.
**Действие:**
- Проверяет состояние турнира (`Created`).
- Проверяет ownership и authorization (через ERC-7857).
- Записывает агента как участника.
- Добавляет `msg.value` к `totalEntryFees`.
- Если набралось `maxSlots`, переводит турнир в статус `Active`. Эмитит `MatchReady`.

#### 3. `placeBet`
**Кто вызывает:** Зритель/Наблюдатель.
**Параметры:** `agentId`.
**Оплата (Value):** Сумма ставки (Native 0G).
**Действие:**
- Проверяет состояние турнира (`Created`).
- Проверяет, что `block.timestamp < config.startTime` (Окно ставок открыто).
- Записывает ставку пользователя в маппинг `bets[msg.sender]`.
- Добавляет `msg.value` к `totalBetsPool`.
- Добавляет `msg.value` к `totalBetsOnAgent[agentId]` (Для расчета коэффициентов).

#### 4. `resolveTournament`
**Кто вызывает:** Кто угодно (Relayer / TEE Orchestrator Watcher).
**Параметры:** `winnerAgentId`, `taskId`, `signature`.
**Действие:**
- Проверяет состояние турнира (`Active`).
- **Восстанавливает подписанта из `signature`** (используя сообщение, включающее адрес текущего турнира, `winnerAgentId` и `taskId`).
- Проверяет, что восстановленный адрес совпадает с `config.refereeTappAddress` (Подтверждение от TEE).
- Устанавливает победителя. Переводит турнир в статус `Finished`.
- Эмитит `TournamentResolved`.

#### 5. `claimRewards`
**Кто вызывает:** 
1. Владелец победившего агента.
2. Зрители, поставившие на победившего агента.
3. Организатор (забирает комиссию).
**Действие:**
- Проверяет состояние (`Finished`).
- Проверяет, не забирал ли пользователь награду.
- Рассчитывает сумму:
  - **Если msg.sender == Организатор:** Рассчитывает комиссию от пула депозитов и пула ставок.
  - **Если msg.sender == Владелец Победителя:** Получает весь `totalEntryFees` (минус комиссия).
  - **Если msg.sender == Зритель (Вариант А):** Рассчитывает долю от `totalBetsPool` пропорционально своей ставке к `totalBetsOnAgent[winnerAgentId]`, забирает выигрыш (минус комиссия).
- Отправляет Native 0G.

## Безопасность и Edge Cases (Ограничения MVP)
1. **Ничья:** Если TEE возвращает ничью (winnerAgentId = 0), нужно реализовать `refund` логику: вернуть депозиты участникам и ставки зрителям (без взимания комиссии).
2. **Reentrancy:** Все функции, отправляющие средства (`claimRewards`, `refund`), обязаны использовать паттерн Checks-Effects-Interactions или `nonReentrant` модификатор.
3. **Таймаут турнира:** Если турнир застрял в состоянии `Created` (не набрал участников) или `Active` (TEE сломался), нужна функция `cancelTournament`, вызываемая владельцем турнира спустя определенный таймаут, для возврата средств.
