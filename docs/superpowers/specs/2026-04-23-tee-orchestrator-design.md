# TEE Chess Orchestrator (Подсистема 2) - Спецификация

**Goal:** Создать TEE-совместимый оркестратор для турниров Unreal AI Tournament (MVP: Chess, 1v1), который управляет развертыванием ИИ-агентов в среде 0G Tapp, проводит матч, фиксирует результаты и отправляет верифицированное доказательство (task_id) в смарт-контракт.

**Architecture:** "Side-by-Side Containers" в 0G Tapp. 
Оркестратор (внешний сервис) реагирует на on-chain события и деплоит `docker-compose.yaml` с 3 независимыми сервисами:
1. `referee` (Python + python-chess): Судья матча.
2. `agent1` (Python): ИИ-агент игрока 1 (изолированный контейнер).
3. `agent2` (Python): ИИ-агент игрока 2 (изолированный контейнер).

**Tech Stack:** 
- Python 3.10+ (Referee & Agents)
- `python-chess` (Игровая логика)
- Docker & Docker Compose (Среда выполнения)
- Bash-скрипты `0g-tapp` (`start_app.sh`, `get_app_log.sh`)
- Node.js/TS (External Watcher & Submitter)

## Механизм шифрования и передачи кода (ERC-7857 + 0G Tapp)

Чтобы код агента оставался приватным, но при этом мог быть исполнен внутри TEE, используется следующий флоу:

1. **Генерация ключей Tapp:** Приложение-судья (Tournament Tapp) имеет свою пару ключей (Public/Private). Public Key известен всем.
2. **Шифрование пользователем:** Владелец ИИ-агента шифрует свой `agent.py` с помощью симметричного ключа. Затем этот симметричный ключ он шифрует с помощью **Public Key турнирного Tapp'а**.
3. **Загрузка в 0G Storage:** Зашифрованный скрипт и зашифрованный симметричный ключ загружаются в 0G Storage. Хэш сохраняется в iNFT (в рамках ERC-7857 это `intelligentData`).
4. **Расшифровка внутри TEE:** 
   - `Referee` запускается внутри TEE и запрашивает приватный ключ приложения через `./get_app_secret_key.sh` (встроенный механизм 0G Tapp).
   - `Referee` скачивает зашифрованные файлы из 0G Storage.
   - Используя приватный ключ приложения, `Referee` расшифровывает симметричные ключи агентов, а затем и сами скрипты `agent.py`.
   - `Referee` монтирует расшифрованные скрипты в защищенный `shared volume` памяти (tmpfs), к которому имеют доступ контейнеры `agent1` и `agent2`.

## Компоненты системы

### 1. External Watcher (Внешний сервис)
Слушает события смарт-контракта и инициирует матчи.
- **Trigger:** Событие `MatchReady(tournamentId, agent1NFT, agent2NFT)`
- **Action:** 
  1. Вытягивает `intelligentDataHash` (хэш в 0G Storage) для обоих агентов из их iNFT.
  2. Генерирует `docker-compose.yaml` (шаблонный) для конкретного матча.
  3. Запускает `./start_app.sh` (из `.references/og-tapp/examples/`) с передачей `docker-compose.yaml` и ключей организатора.
  4. Получает `task_id` от Tapp.
  5. Мониторит статус выполнения. По завершении - дергает `./get_app_log.sh` для `referee`.
  6. Отправляет транзакцию `SubmitMatchResult(tournamentId, winnerAddress, taskId, signature)` в контракт.

### 2. TEE Referee (Внутри Tapp)
Главный контейнер, управляющий игрой.
- **Инициализация:** 
  - Получает хэши агентов через переменные окружения.
  - Скачивает зашифрованные скрипты из 0G Storage.
  - Вызывает локальный API Tapp (`get_app_secret_key.sh`) для получения приватного ключа.
  - Расшифровывает скрипты и сохраняет их в `shared volume` (в оперативной памяти).
  - Ждет, пока `agent1` и `agent2` поднимут свои HTTP-серверы (healthcheck).
- **Игровой цикл:**
  - Использует `python-chess` для ведения состояния доски (`chess.Board()`).
  - Отправляет HTTP POST `http://agent1:8080/move` (или agent2), передавая текущий FEN: `{"fen": "rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq - 0 1"}`.
  - Ожидает ответ `{"move": "e2e4"}`. **Лимит ожидания - 60 секунд.**
  - Применяет ход к `chess.Board()`. Если ход невалидный (не по правилам) или таймаут - агенту засчитывается техническое поражение.
  - Цикл продолжается до мата, пата, ничьей.
- **Финализация:**
  - Выводит результат (победитель, PGN, причины завершения) в stdout, который собирается в логи Tapp.
  - Завершает работу (Exit 0).

### 3. AI Agent (Внутри Tapp)
Контейнер участника, запускающийся **как отдельный сервис** в docker-compose.
- **Boot:** Контейнер стартует с базовым образом-оберткой (HTTP-сервером). Он ожидает, когда `Referee` положит расшифрованный `agent.py` в `shared volume`, после чего загружает этот модуль.
- **Интерфейс:** Простой HTTP-сервер (Flask/FastAPI).
- **Код:** `agent.py` участника. Обязан реализовать функцию `def get_move(fen: str) -> str:`.
- **Ограничения:** В `docker-compose.yaml` для сети агентов закрыт выход в интернет (internal network). Только ответ на запросы от `Referee`.

## Структура директорий (Планируемая)
```text
tee-orchestrator/
├── watcher/                # Внешний Node.js/TS сервис
│   ├── index.ts
│   └── tapp-client.ts      # Обертка над bash-скриптами 0G Tapp

│   ├── Dockerfile
│   ├── main.py             # Логика игры
│   ├── storage.py          # Работа с 0G Storage и расшифровкой
│   └── requirements.txt
└── agent-base/             # Базовый образ агента
    ├── Dockerfile
    ├── server.py           # HTTP обертка, подгружающая /shared/agent.py
    └── requirements.txt
```
